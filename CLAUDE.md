# 프로젝트 규칙

## 작업 완료 절차 (필수)

모든 코드 변경 작업 후 반드시 아래 순서를 따른다:

1. **커밋** — 변경 내용을 명확한 메시지로 커밋
2. **버전 업** — `js/app.js`의 `APP_VERSION`을 올린다 (예: 0.68 → 0.69)
   - build 날짜도 오늘 날짜로 갱신 (예: `build 20260424a`)
3. **버전 커밋** — 버전 변경을 별도 커밋으로 남긴다
4. **PR 생성 → main 병합** — PR을 만들고 즉시 squash merge
5. 사용자에게 완료 보고 시 **적용된 버전 번호**를 명시

> 사용자가 수정 결과를 앱에서 바로 확인할 수 있도록,
> 모든 작업은 반드시 main 병합 + 버전 표시까지 완료한다.
